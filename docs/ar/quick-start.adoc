ابدأ تشغيل WIS2 Downloader وتنزيل بياناتك الأولى في غضون دقائق.

== المتطلبات الأساسية

* https://docs.docker.com/get-docker/[Docker] 20.10+
* https://docs.docker.com/compose/install/[Docker Compose] 2.0+
* اتصال شبكي بـ Global Broker الخاص بـ WIS2 (المنفذ 443)

== 1. التنزيل والاستخراج

نزّل أرشيف https://github.com/World-Meteorological-Organization/wis2downloader/releases/latest[أحدث إصدار] واستخرج محتوياته:

[source,bash]
----
tar -xzf wis2downloader-*.tar.gz
cd wis2downloader-*
----

TIP: للمتابعة مع أحدث إصدار تطويري، استنسخ المستودع: `git clone https://github.com/World-Meteorological-Organization/wis2downloader.git`

== 2. التهيئة

شغّل سكريبت الإعداد لإنشاء ملف `.env` بمفاتيح سرية عشوائية:

[source,bash]
----
bash setup.sh
----

يقوم هذا السكريبت بنسخ `default.env` إلى `.env` وتوليد قيم عشوائية لـ `FLASK_SECRET_KEY` و`REDIS_PASSWORD`. راجع ملف `.env` وعدّل الإعدادات إذا لزم الأمر — القيم الافتراضية مناسبة للنشر على جهاز واحد.

IMPORTANT: احتفظ بملف `.env` في مكان آمن. يحتوي على بيانات اعتماد حساسة ويجب عدم إدراجه في نظام إدارة الإصدارات.

== 3. تثبيت مكوّن Loki الإضافي لـ Docker

يستخدم WIS2 Downloader https://grafana.com/docs/loki/latest/send-data/docker-driver/[مشغّل Docker الخاص بـ Loki] لإرسال السجلات. ثبّته قبل تشغيل المكدّس:

[source,bash]
----
ARCH=$(uname -m | sed 's/x86_64/amd64/;s/aarch64/arm64/')
docker plugin install grafana/loki-docker-driver:3.6.7-${ARCH} --alias loki --grant-all-permissions
----

تحقق من تفعيل المكوّن الإضافي:

[source,bash]
----
docker plugin ls
----

يجب أن ترى `loki:latest` مع `ENABLED` بقيمة `true`.

NOTE: إذا ظهرت رسالة خطأ أثناء التثبيت، تحقق من معمارية جهازك باستخدام `uname -m`. أكثر القيم شيوعًا هي `x86_64` (amd64) و`aarch64` (arm64).

== 4. إعداد مجلد التنزيلات

تعمل الحاويات بصلاحيات مستخدم غير جذري (UID 10001، GID 988). على نظام Linux، يجب إعداد مجلد التنزيلات قبل البدء:

[source,bash]
----
sudo groupadd -g 988 wis2
sudo useradd -u 10001 -g wis2 -M -s /usr/sbin/nologin wis2
mkdir -p downloads
sudo chown wis2:wis2 downloads
----

NOTE: على نظامَي macOS وWindows (Docker Desktop)، هذه الخطوة غير مطلوبة — يتولى Docker Desktop إدارة صلاحيات الأحجام تلقائيًا.

إن تعذّر عليك إنشاء مستخدم مطابق، راجع قسم link:admin-guide.html#_volume_permissions[صلاحيات الأحجام] في دليل الإدارة للاطلاع على البدائل المستندة إلى ACL.

== 5. تشغيل الخدمات

[source,bash]
----
docker compose up -d
----

يُشغّل هذا الأمر جميع الخدمات: مدير الاشتراكات، مشتركو MQTT (Global Brokers لفرنسا والبرازيل)، عمّال Celery للتنزيل، Redis، Prometheus، Grafana، وLoki.

تحقق من تشغيل جميع الخدمات:

[source,bash]
----
docker compose ps
----

انتظر حتى يجتاز `subscription-manager` فحص الصحة (نحو 30 ثانية)، ثم تأكد من تشغيله:

[source,bash]
----
curl http://localhost:5002/health
----

== 6. إنشاء اشتراك

=== عبر واجهة الويب (موصى به)

افتح http://localhost:8080 في متصفحك.

استخدم *عرض الكتالوج* أو *عرض الشجرة* لاستعراض مجموعات بيانات WIS2 المتاحة، واختر موضوعًا، وحدد مجلد الحفظ، ثم انقر على *اشتراك*.

=== عبر API

[source,bash]
----
curl -X POST http://localhost:5002/subscriptions \
  -H "Content-Type: application/json" \
  -d '{
    "topic": "cache/a/wis2/+/data/core/weather/surface-based-observations/#",
    "target": "surface-obs"
  }'
----

يُنشئ هذا اشتراكًا بجميع رصد السطح من أي مركز بيانات ويحفظ الملفات في `downloads/surface-obs/`.

== 7. عرض التنزيلات

تظهر الملفات في مجلد `downloads/` مرتّبةً حسب التاريخ:

[source,bash]
----
ls -la downloads/surface-obs/
----

راقب نشاط التنزيل في Grafana على http://localhost:3000 (بيانات الاعتماد الافتراضية: `admin` / `admin`).

IMPORTANT: غيّر كلمة المرور الافتراضية لـ Grafana بعد تسجيل الدخول الأول.
